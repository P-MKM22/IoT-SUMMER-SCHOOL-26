# Q 07 – Collaborative Fork & Pull Request

## Overview

This project demonstrates a collaborative Git and GitHub workflow by contributing to another developer's repository using **Forks** and **Pull Requests (PRs)**. A modified Arduino program (`blink_modified.ino`) was added to the forked repository, committed using the Conventional Commits format, and submitted as a Pull Request for review. The Pull Request was successfully reviewed, merged by the repository owner, and the fork was synchronized with the upstream repository.

This exercise provides practical experience with collaborative software development using GitHub.

---

## Objectives

- Understand the concept of forking a GitHub repository.
- Learn how to contribute to another developer's project.
- Create and commit new source code.
- Submit a professional Pull Request.
- Experience the code review and merge process.
- Synchronize a fork with the upstream repository.

## Task Description

The objective of this task was to contribute to another student's GitHub repository using the standard GitHub collaboration workflow.

The following implementation was completed:

- Forked the partner's **iot-summer-school-2026** repository.
- Created a new Arduino program named **blink_modified.ino**.
- Implemented the required LED blinking logic.
- Committed the changes using the Conventional Commits format.
- Submitted a Pull Request with a proper title and description.
- The Pull Request was reviewed and merged by the repository owner.
- Finally, the fork was synchronized with the upstream repository.

---

## Program Description

The Arduino program performs the following sequence:

- Blinks the LED **three times rapidly**.
- Waits for **2 seconds**.
- Repeats the process continuously.

This modification satisfies the requirements specified in the assignment.
