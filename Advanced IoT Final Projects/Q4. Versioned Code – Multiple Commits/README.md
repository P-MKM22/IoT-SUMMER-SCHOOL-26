# Q 04 вЂ“ Versioned Development of an LED Blinking Project

## Overview

This question demonstrates the development of a simple Arduino-based LED blinking project through multiple software versions. Instead of implementing all features at once, the project evolves incrementally by introducing new functionality in each version while preserving the previous implementation.

The objective is to understand how software projects are developed, documented, and maintained using proper version control practices. Every version represents a milestone in the development process and is accompanied by its own source code, documentation, project assets, and Wokwi simulation.

---

## Task Objectives

This question focuses on:

- Developing an Arduino project through multiple versions.
- Implementing one enhancement in each version.
- Maintaining proper project documentation.
- Following Conventional Commits for every update.
- Organizing the repository using a professional folder structure.

## Version Control

Each project version has been committed using the **Conventional Commits** standard to maintain a structured and meaningful development history.

Examples:

```text
feat: Add Version 1.0 LED Blinking Program
feat: Add Serial Monitor Blink Counter
feat: Add Potentiometer Controlled Blink Speed
docs: Add Program Header and Documentation
```
