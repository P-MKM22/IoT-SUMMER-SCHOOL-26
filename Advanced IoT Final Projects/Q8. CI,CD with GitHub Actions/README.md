# Q 08 – CI/CD with GitHub Actions

## Overview

This project demonstrates the implementation of a basic **Continuous Integration (CI)** workflow using **GitHub Actions**. The workflow is designed to automatically execute a Python script whenever new code is pushed to the repository. The Python script recursively scans the repository, counts all Arduino (`.ino`) files, and prints the total count in the workflow logs.

This task introduces workflow automation using GitHub Actions and highlights how repetitive development tasks can be automated efficiently.

---

## Objectives

- Understand the fundamentals of Continuous Integration (CI).
- Learn how to create and configure GitHub Actions workflows.
- Automate repository tasks using Python.
- Execute workflows automatically on every push.
- Display repository information through workflow logs.

## Project Description

The project consists of two major components:

### 1. Python Script (`count_ino.py`)

The Python script performs the following tasks:

- Searches the repository recursively.
- Identifies every Arduino `.ino` file.
- Counts the total number of `.ino` files.
- Displays the total count.
- Prints the relative path of each `.ino` file found.

---

### 2. GitHub Actions Workflow (`lint.yml`)

The workflow automatically performs the following operations whenever code is pushed to the repository:

1. Checks out the repository.
2. Installs Python.
3. Executes the Python script.
4. Displays the results in the GitHub Actions logs.

This automation eliminates the need to manually execute the script after every update.
