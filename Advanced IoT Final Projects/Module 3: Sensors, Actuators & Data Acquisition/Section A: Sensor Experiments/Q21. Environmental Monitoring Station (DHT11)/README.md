# Q21 – Environmental Monitoring Station (DHT11)

## Overview

This project implements a mini Environmental Monitoring Station using an ESP32 and a DHT22 sensor (used in Wokwi as a substitute for the DHT11). The system measures temperature and humidity every two seconds, displays the readings in CSV format on the Serial Monitor, and indicates environmental conditions using Red and Green LEDs.

> **Note:** The assignment specifies the DHT11 sensor. Since Wokwi currently provides the DHT22 sensor, it has been used for simulation. The implementation logic remains identical.

---

## Objectives

- Interface a DHT sensor with the ESP32.
- Measure temperature in Celsius and Fahrenheit.
- Measure relative humidity.
- Display readings every two seconds in CSV format.
- Indicate abnormal environmental conditions using LEDs.
- Store sample sensor readings in a CSV file.

## Working Principle

The DHT sensor continuously measures temperature and humidity. Every two seconds, the ESP32 reads the sensor values, converts the temperature from Celsius to Fahrenheit, and displays the readings on the Serial Monitor in CSV format.

The measured values are compared with predefined thresholds:

- Temperature > 35°C
- Humidity > 80%

If either condition is met, the Red LED turns ON to indicate abnormal environmental conditions. Otherwise, the Green LED remains ON to indicate normal conditions.

## Circuit Connections

| ESP32 Pin | Connected Component |
|------------|--------------------|
| 3.3V | DHT22 VCC |
| GND | DHT22 GND |
| GPIO15 | DHT22 DATA |
| GPIO18 | Red LED |
| GPIO19 | Green LED |
| GND | LED Cathodes (via 220 Ω resistors) |

## Execution Steps

### Step 1: Build the Circuit

Connect the ESP32, DHT22 sensor, Red LED, and Green LED according to the circuit diagram.

### Step 2: Open the Wokwi Project

Open the simulation using the link provided below.

### Step 3: Verify the Source Code

Open the Arduino sketch and verify that the GPIO pin assignments match the circuit.

### Step 4: Compile the Program

Compile the sketch to ensure there are no errors.

### Step 5: Start the Simulation

Click **Start Simulation**.

The ESP32 begins reading sensor values every two seconds.

### Step 6: Open the Serial Monitor

Set the baud rate to **115200**.

Observe the CSV output.

Example:

```csv
timestamp (s),temp_C (°C),temp_F (°F),humidity (%)
0,29.50,85.10,58.20
2,29.70,85.46,58.40
4,29.90,85.82,58.60
```

### Step 7: Test Normal Conditions

Set:

- Temperature = 30°C
- Humidity = 60%

Expected Result:

- Green LED ON
- Red LED OFF

### Step 8: Test High Temperature

Increase the temperature above **35°C**.

Expected Result:

- Red LED ON
- Green LED OFF

### Step 9: Test High Humidity

Increase humidity above **80%**.

Expected Result:

- Red LED ON
- Green LED OFF

### Step 10: Save CSV Data

Copy at least five consecutive readings from the Serial Monitor and save them as:

```text
Data/sample_readings.csv
```

Commit the CSV file to the GitHub repository.
