# Q3 – Branching Strategy

## Task overview

Version control is an essential part of modern software development. This project demonstrates the implementation of Git branching using GitHub. Multiple branches were created to simulate a collaborative development workflow, allowing independent feature development before merging changes into the main branch.

---

## Objectives

- Understand the concept of Git branches.
- Create and manage multiple branches.
- Learn branch-based development workflow.
- Merge changes into the main branch using GitHub.
- Gain practical experience with version control.
