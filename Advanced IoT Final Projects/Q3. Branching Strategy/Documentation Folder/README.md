# Documentation – Q3 Branching Strategy

## Introduction

This documentation describes the implementation of Git branching as part of the IoT and Drone Building assignment. The objective was to understand how Git branches are created, managed, and merged using GitHub.

---

## Objective

- Understand the concept of Git branches.
- Create multiple development branches.
- Learn the branch-based development workflow.
- Merge changes into the main branch using a pull request.
- Gain practical experience with GitHub version control.

---

## Branches created

### Main

The primary branch that contains the stable version of the repository.

### Dev

The development branch used to test and integrate changes before merging into the main branch.

### Sensor-Module

A feature branch created for sensor-related development and experimentation.

### Wireless-Module

A feature branch created for wireless communication-related development.

---

## File created

### Notes.md

The `Notes.md` file was created in the Dev branch to describe the purpose of each branch before merging it into the main branch.

---

## Workflow followed

1. Created the Dev branch.
2. Created the Sensor-Module branch.
3. Created the Wireless-Module branch.
4. Added the `Notes.md` file to the Dev branch.
5. Committed the changes.
6. Created and merged the pull request into the main branch.
7. Verified the successful merge.

After completing this assignment, I learned how to:

This assignment provided practical experience with GitHub branching and version control.
