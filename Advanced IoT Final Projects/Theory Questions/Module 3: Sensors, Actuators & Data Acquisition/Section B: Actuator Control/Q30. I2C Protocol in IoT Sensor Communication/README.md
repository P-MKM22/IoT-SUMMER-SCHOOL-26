# Introduction

The **Inter-Integrated Circuit (I2C)** protocol is a synchronous serial communication protocol developed by Philips (now NXP). It is widely used in IoT and embedded systems because it allows multiple sensors and devices to communicate with a microcontroller using only **two communication wires**, making the circuit simple and efficient.

---

# What is I2C Protocol?

I2C (Inter-Integrated Circuit) is a communication protocol that enables a **master device** (such as an Arduino UNO or ESP32) to communicate with one or more **slave devices** (such as sensors and displays) over a shared two-wire bus.

### Features

- Uses only **two communication lines**.
- Supports multiple devices on the same bus.
- Each device has a unique address.
- Simple wiring and low pin usage.
- Commonly used in IoT sensor networks.

---

# SDA and SCL Lines

| Line | Full Form | Function |
|------|-----------|----------|
| **SDA** | Serial Data Line | Transfers data between the master and slave devices. |
| **SCL** | Serial Clock Line | Carries the clock signal generated by the master to synchronize communication. |

### Arduino UNO I2C Pins

| Pin | Function |
|------|----------|
| **A4** | SDA |
| **A5** | SCL |

---

# How Does I2C Addressing Work?

Each I2C device is assigned a unique **7-bit** (or sometimes **10-bit**) address.

### Communication Process

1. The master device starts communication.
2. It sends the address of the target slave device.
3. Every device on the I2C bus receives the address.
4. Only the device with the matching address responds.
5. Data is exchanged between the master and the selected slave.
6. The communication ends with a stop condition.

This addressing mechanism allows multiple I2C devices to share the same SDA and SCL lines without interfering with each other.

# Common IoT Sensors Using I2C

| Sensor | Application |
|---------|-------------|
| **MPU6050** | Motion sensing using accelerometer and gyroscope. |
| **BMP280** | Measures atmospheric pressure, altitude, and temperature. |
| **BME280** | Measures temperature, humidity, and atmospheric pressure. |

Other common I2C devices include OLED displays (SSD1306), BH1750 light sensors, DS3231 RTC modules, and ADS1115 ADC modules.

---

# Practical IoT Example

A **smart weather monitoring system** uses an Arduino UNO connected to a **BME280 sensor** through the I2C bus.

- The Arduino acts as the **master**.
- The BME280 acts as the **slave**.
- Communication occurs through the **SDA (A4)** and **SCL (A5)** lines.
- The Arduino reads temperature, humidity, and pressure data and sends it to a cloud platform or displays it on an OLED screen.

---

# Advantages of I2C

- Requires only two communication wires.
- Supports multiple devices on the same bus.
- Reduces wiring complexity.
- Saves microcontroller I/O pins.
- Easy to interface with sensors and displays.
- Widely supported in IoT and embedded systems.

---

# Conclusion

The I2C protocol is an efficient communication method for connecting multiple sensors and peripheral devices to a microcontroller using only two wires: **SDA** for data transfer and **SCL** for clock synchronization. Through unique device addressing, multiple sensors can share the same communication bus, making I2C an ideal choice for compact, reliable, and scalable IoT applications.
